Checklist

이해도 체크리스트

  • async 함수를 호출할 때 반드시 await를 붙여야 하는 이유를 안다
  • try await URLSession.shared.data(from:url)의 반환 타입을 설명할 수 있다
  • CodingKeys enum으로 JSON 키와 Swift 프로퍼티를 매핑하는 방법을 구현할 수 있다
  • @MainActor.run { } 이 필요한 이유를 설명할 수 있다
  • .task { await service.fetch(city:) } 가 언제 실행되는지 안다
  • weatherData를 [UUID: WeatherResponse] 딕셔너리로 캐싱하는 이유를 이해한다

핵심 코드 패턴

1. async/await 네트워크 요청

func fetch(city: City) async { var comps = URLComponents(string: "https://api.open-meteo.com/v1/forecast")! comps.queryItems = [ .init(name: "latitude", value: String(city.lat)), .init(name: "longitude", value: String(city.lon)), .init(name: "current", value: "temperature_2m,weathercode"), .init(name: "hourly", value: "temperature_2m,weathercode"), ] let (data, _) = try! await URLSession.shared.data(from: comps.url!) let response = try! JSONDecoder().decode(WeatherResponse.self, from: data) await MainActor.run { weatherData[city.id] = response // UI 스레드에서 업데이트 } }

2. CodingKeys 매핑

struct CurrentWeather: Codable { let temperature: Double let weathercode: Int enum CodingKeys: String, CodingKey { case temperature = "temperature_2m" // JSON 키 → Swift 이름 case weathercode = "weathercode" } }
