Topic 01

Computed Property

A property that calculates its value instead of storing it
Swift
struct Rectangle {
    var width: Double
    var height: Double

    // Computed Property — if only get exists, get can be omitted
    var area: Double {
        width * height
    }

    // get + set
    var perimeter: Double {
        get { (width + height) * 2 }
        set { width = newValue / 2 - height }
    }
}

var rect = Rectangle(width: 10, height: 5)
print(rect.area)      // 50.0
print(rect.perimeter) // 30.0

// A common pattern in SwiftUI
struct UserProfile {
    var firstName: String
    var lastName: String

    // Combine without storing
    var fullName: String { "\(firstName) \(lastName)" }

    // Condition check
    var isNameValid: Bool { !firstName.isEmpty && !lastName.isEmpty }

    // Formatting
    var initials: String {
        let f = firstName.first.map(String.init) ?? ""
        let l = lastName.first.map(String.init) ?? ""
        return f + l
    }
}
Think about it
  • When should you use a computed property vs. a function? What about when the computation is expensive?
  • Why can you omit get { } in a read-only computed property?
  • How can you rename newValue in a set block?
Read-only computed property
If only a get { } block exists, it can be omitted. var area: Double { width * height }
get + set
Access the new value with newValue in set. Useful for bidirectional conversions (e.g., Celsius ↔ Fahrenheit)
Function vs computed property
No parameters and no side effects → computed property. Has side effects or requires parameters → function
Topic 02

Property Observer — willSet / didSet

Run code before or after a property value changes
Swift
class StepCounter {
    var steps: Int = 0 {
        willSet {
            // Called just before the change — access new value via newValue
            print("About to change to \(newValue)")
        }
        didSet {
            // Called right after the change — access previous value via oldValue
            print("Changed from \(oldValue) to \(steps)")
            if steps >= 10000 {
                print("Goal reached!")
            }
        }
    }
}

// A practical pattern in SwiftUI ViewModels
class CartViewModel: ObservableObject {
    @Published var items: [CartItem] = [] {
        didSet {
            // Auto-save whenever items change
            save()
        }
    }

    @Published var searchText: String = "" {
        didSet {
            // Refresh results when search text changes (simple filter)
            filterItems()
        }
    }

    private func save() { /* Save to UserDefaults */ }
    private func filterItems() { /* Filter logic */ }
}

// Note: you cannot attach didSet directly to @State
// → Use the .onChange(of:) modifier instead
struct ContentView: View {
    @State private var count = 0

    var body: some View {
        Stepper("\(count)", value: $count)
            .onChange(of: count) { old, new in
                print("\(old) → \(new)")
            }
    }
}
Think about it
  • Can you modify the value in willSet? What about in didSet?
  • Is didSet called when you assign a value to a property inside init?
  • Why can't you use didSet with SwiftUI's @State?
willSet
Called just before the value changes. Access the new value via newValue. You cannot modify the value here
didSet
Called right after the value changes. Access the old value via oldValue. Modifying the same property inside didSet runs only once without recursion
Not called during init
Property observers are not triggered during initializer assignments. This is by design
Topic 03

struct vs class — Value Type vs Reference Type

The most important type choice in Swift
Swift
// struct — value type (copy)
struct PointS {
    var x: Int
    var y: Int
}

var a = PointS(x: 1, y: 2)
var b = a        // Creates a copy
b.x = 99
print(a.x) // 1  ← a is unchanged

// class — reference type (points to the same object)
class PointC {
    var x: Int
    var y: Int
    init(x: Int, y: Int) { self.x = x; self.y = y }
}

var c = PointC(x: 1, y: 2)
var d = c        // Points to the same object
d.x = 99
print(c.x) // 99 ← c is also changed!

// When to use which in SwiftUI
// struct: data models, View (SwiftUI Views are always structs)
// class: ObservableObject, network managers, singletons

// Modifying self in a struct method — mutating
struct Counter {
    var count = 0

    mutating func increment() {
        count += 1  // Compile error without mutating
    }

    mutating func reset() {
        count = 0
    }
}
Think about it
  • Why are SwiftUI Views structs? What problems would arise if they were classes?
  • Can you modify properties of a class instance declared with let a = SomeClass()?
  • What is Copy-on-Write, and how do Swift's Array/Dictionary leverage it?
struct = value type
Copied on assignment/passing. Independent state. Thread-safe. Ideal for SwiftUI Views and data models
class = reference type
Shares the same instance on assignment/passing. Supports inheritance. Used for ObservableObject and shared state
mutating
Required when a struct method modifies self (the instance). Also applies to enum methods
Topic 04

Access Control

Deciding what to expose and what to hide from the outside
Swift
// Access levels (from broadest to narrowest)
// open > public > internal (default) > fileprivate > private

class BankAccount {
    // private: accessible only within this class
    private var balance: Double = 0

    // internal (default): accessible within the same module
    var owner: String

    // public: accessible from external modules as well
    public var accountNumber: String

    init(owner: String, number: String) {
        self.owner = owner
        self.accountNumber = number
    }

    // private(set): readable from anywhere, writable only internally
    private(set) var transactionCount = 0

    func deposit(_ amount: Double) {
        guard amount > 0 else { return }
        balance += amount
        transactionCount += 1
    }

    func withdraw(_ amount: Double) -> Bool {
        guard amount <= balance else { return false }
        balance -= amount
        transactionCount += 1
        return true
    }
}

// A practical pattern in SwiftUI ViewModels
class CounterViewModel: ObservableObject {
    @Published private(set) var count = 0  // Read-only from outside

    func increment() { count += 1 }   // Only modifiable internally
    func decrement() { count -= 1 }
    func reset()     { count = 0  }
}
Think about it
  • What is the difference between private and fileprivate? Can you access a private member from an extension in the same file?
  • Why use the pattern @Published private(set) var count?
  • What problems arise if you never use access control? What does it mean for team projects?
private
Accessible only within the declaring type + extensions in the same file. The most commonly used level
internal (default)
Applied when no keyword is specified. Accessible throughout the entire module (app target)
private(set)
Anyone can read, only internal code can write. Commonly used to protect @Published properties in ViewModels
Topic 05

init — Initializer Patterns

How to safely initialize instances
Swift
// struct: automatically generates a memberwise init
struct Point {
    var x: Double
    var y: Double
    // Swift automatically generates init(x:y:)
}
let p = Point(x: 1, y: 2)

// Adding a custom init removes the memberwise init
// Separate it into an extension to keep both
extension Point {
    init(value: Double) {
        self.x = value
        self.y = value
    }
}

// class: you must write init yourself
class Person {
    let name: String
    var age: Int
    var email: String?   // Optional doesn't need to be set in init

    // Designated initializer
    init(name: String, age: Int) {
        self.name = name
        self.age = age
    }

    // Convenience initializer — reuses another init
    convenience init(name: String) {
        self.init(name: name, age: 0)
    }
}

// Failable init — initialization that can fail
struct Color {
    let hex: String

    init?(hex: String) {  // Note the ?
        guard hex.hasPrefix("#"), hex.count == 7 else {
            return nil  // Initialization fails → returns nil
        }
        self.hex = hex
    }
}

let valid = Color(hex: "#FF6B2B")   // Optional
let invalid = Color(hex: "bad")     // nil
Think about it
  • You added a custom init to a struct and existing code broke with a compile error. Why? How do you fix it?
  • Why must convenience init always call self.init(...)?
  • What is the difference between using init? (failable) and throwing an error with throws?
Memberwise init (auto-generated for structs)
When no custom init exists, an init with parameters for all stored properties is auto-generated. Add custom inits in an extension to keep both
convenience init
A convenience initializer in a class that reuses another designated init. Must call an init from the same class
Failable init — init?
Returns nil if initialization conditions are not met. Result is Optional. Commonly used with URL, type conversions, etc.
Topic 06

Type Casting — as? / as! / is

How to check and convert types at runtime
Swift
class Animal { var name: String; init(_ n: String) { name = n } }
class Dog: Animal { func bark() { print("Woof!") } }
class Cat: Animal { func meow() { print("Meow") } }

let animals: [Animal] = [Dog("Buddy"), Cat("Whiskers"), Dog("Max")]

// is — type check
for animal in animals {
    if animal is Dog {
        print("\(animal.name) is a dog")
    }
}

// as? — safe downcasting (returns Optional)
for animal in animals {
    if let dog = animal as? Dog {
        dog.bark()  // Can call Dog-specific methods
    }
}

// as! — forced downcasting (crashes on failure)
let firstAnimal = animals[0]
let dog = firstAnimal as! Dog  // Use only when you're certain

// Upcasting with as (always succeeds, no Optional needed)
let d = Dog("Puppy")
let a: Animal = d as Animal

// Practical: Codable + Any type handling
let json: [String: Any] = ["count": 5, "name": "test"]
if let count = json["count"] as? Int {
    print(count) // 5
}

// switch + pattern matching
for animal in animals {
    switch animal {
    case let dog as Dog:
        dog.bark()
    case let cat as Cat:
        cat.meow()
    default:
        break
    }
}
Think about it
  • What problems arise from overusing as! instead of as??
  • Which is safer — upcasting (child → parent) or downcasting (parent → child) — and why?
  • Why do you need to use AnyView instead of [any View] in SwiftUI?
as? — conditional cast
Returns nil on failure. Used safely with if let. The most commonly used form in practice
as! — forced cast
Use only when the type is certain. Crashes at runtime on failure. When in doubt, always use as?
is — type check
Returns Bool. Also used with the case let x as Type pattern in switch statements
Topic 07

static · lazy · inout

Three keywords that make your code more flexible
Swift
// static — members that belong to the type, not an instance
struct AppConfig {
    static let baseURL = "https://api.example.com"
    static let timeout: TimeInterval = 30

    static func makeURL(_ path: String) -> URL? {
        URL(string: baseURL + path)
    }
}

AppConfig.baseURL  // Access without an instance
AppConfig.makeURL("/posts")

// Singleton pattern
class NetworkManager {
    static let shared = NetworkManager()
    private init() {}  // Prevents external creation
}

// lazy — initialized on first access
class DataManager {
    // Defer expensive initialization until first use
    lazy var cache: [String: Data] = [:]

    lazy var formatter: DateFormatter = {
        let f = DateFormatter()
        f.dateStyle = .medium
        f.locale = Locale(identifier: "en_US")
        return f
    }()
}

// inout — directly modify a variable outside the function
func doubleValue(_ value: inout Int) {
    value *= 2
}

var number = 5
doubleValue(&number)  // & is required
print(number) // 10

// swap implementation
func swap(_ a: inout T, _ b: inout T) {
    let temp = a
    a = b
    b = temp
}
Think about it
  • What is the difference between the static and class keywords? Can you use class in a struct?
  • Why does using lazy var in a struct require mutating?
  • What is the difference between inout and returning the modified value? Which is more Swift-like?
static
Properties/methods that belong to the type itself. Accessed without an instance. Used for constants, utility functions, and singletons
lazy var
Initialized on first access. Ideal for expensive objects (DateFormatter, large arrays). Cannot be used with let
inout
Lets a function modify the parameter directly. Requires & at the call site. From a functional programming perspective, returning values is preferred
Topic 08

Patterns You'll See in Practice

How the grammar you learned combines in real code
Swift
// A real-world ViewModel — using everything we've learned
@MainActor
class ProductViewModel: ObservableObject {

    // private(set): external read OK, write X
    @Published private(set) var products: [Product] = []
    @Published private(set) var isLoading = false

    // computed property: derived data
    var discountedProducts: [Product] {
        products.filter { $0.discount > 0 }
    }

    var totalCount: Int { products.count }

    // static: shared constants
    static let maxItemsPerPage = 20

    // lazy: created only on first use
    private lazy var cache: [Int: Product] = [:]

    // didSet: cache synchronization
    // (used internally when products are updated)

    func load() async {
        isLoading = true
        defer { isLoading = false }  // defer: always runs when the block exits
        do {
            products = try await ProductService.fetchAll()
        } catch {
            print(error.localizedDescription)
        }
    }
}

// struct model — value type
struct Product: Identifiable, Codable {
    let id: Int
    let name: String
    let price: Double
    var discount: Double = 0

    // computed property
    var finalPrice: Double { price * (1 - discount) }
    var isOnSale: Bool { discount > 0 }

    // static factory method
    static func mock() -> Product {
        Product(id: 0, name: "Test Product", price: 9900)
    }
}
Think about it
  • What role does defer play in this code? How is it different from just writing isLoading = false at the end of the function?
  • Why is Product a struct and ProductViewModel a class?
  • What is the difference between @Published private(set) and @Published private?
defer
Runs when the block exits (whether by return, throw, or normal completion). Useful for cleanup code (isLoading = false, closing files)
Static factory method
The static func mock() pattern. Useful for generating sample data in Previews and tests
Connecting all the grammar
Each concept is learned independently, but in real code they combine together. Reading practice comes before writing practice
