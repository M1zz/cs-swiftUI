Topic 01

SwiftUI Rendering Loop

State Change → View Re-creation Mechanism
Swift 
struct CounterView: View {
    @State private var count = 0

    var body: some View {
        VStack(spacing: 20) {
            Text("\(count)")
                .font(.system(size: 60, weight: .bold))

            Button("+ 1") {
                count += 1
            }
            .buttonStyle(.borderedProminent)
        }
    }
}
Think About It
  • A SwiftUI View is a struct — why can't you directly mutate a property inside a struct?
  • Why does a value declared with @State persist even when the View is re-created?
  • When you tap the button and change count, what happens inside SwiftUI step by step?
A View is a struct — it's immutable
SwiftUI Views are value types. When state changes, a new View is created. The existing View is not modified.
Why changing a plain var doesn't work
To mutate a property inside a struct, you need mutating. The body property is not mutating.
Where State is stored
@State stores values in SwiftUI's external storage, outside the View. Values persist across View re-creations.
Topic 02

@State · @Binding

Ownership vs. Sharing
Swift 
struct ParentView: View {
    @State private var name = ""

    var body: some View {
        VStack {
            Text("Name: \(name)")
            ChildView(name: $name)
        }
    }
}

struct ChildView: View {
    @Binding var name: String

    var body: some View {
        TextField("Enter name", text: $name)
            .textFieldStyle(.roundedBorder)
    }
}
Think About It
  • What is the ownership difference between @State and @Binding? Who "owns" the value?
  • What does the $ symbol mean when passing a parent's @State to a child?
  • Why shouldn't a child View declare its own @State for shared data?
@State — Owned by the View
Declared as private. The View is the sole owner of the state. Changes trigger automatic re-rendering.
@Binding — Passes a reference
Passed via $state. The child can read and write the parent's State. It's a reference, not a value.
Unidirectional Data Flow
State flows from top to bottom. Views only render state — the principle is that they don't own it.

Topic 05

List + ForEach + Identifiable

Rendering Arrays as Lists
Swift 
struct Fruit: Identifiable {
    let id = UUID()
    let name: String
    let emoji: String
}

struct FruitList: View {
    @State private var fruits = [
        Fruit(name: "Apple", emoji: "🍎"),
        Fruit(name: "Banana", emoji: "🍌"),
        Fruit(name: "Grape", emoji: "🍇"),
    ]

    var body: some View {
        List {
            ForEach(fruits) { fruit in
                HStack {
                    Text(fruit.emoji)
                    Text(fruit.name)
                }
            }
            .onDelete { fruits.remove(atOffsets: $0) }
        }
    }
}
Think About It
  • Why must data passed to ForEach conform to Identifiable?
  • When do you use id: \.self, and when do you use Identifiable?
  • What is the difference between List and ScrollView + ForEach?
  • What is the IndexSet received in .onDelete(perform:)?
List { ForEach(items) }
Renders each element of an array as a cell. The SwiftUI replacement for UITableView.
Identifiable Protocol
Requires an id property. Allows SwiftUI to distinguish each item and re-render only what's needed.
onDelete(perform:)
Swipe to delete. Passes an IndexSet with the indices of items to remove.

Think About It
  • How do NavigationLink and .sheet() differ in their screen transition approach?
  • Why is isPresented in .sheet(isPresented:) managed with an @State Bool?
  • What is the difference between NavigationStack and the (deprecated) NavigationView?
NavigationStack + NavigationLink
Stack-based navigation. Use .navigationTitle(), .navigationBarItems().
.sheet(isPresented: $show)
Show/hide a modal with a Bool @State. A sheet that slides up from the bottom.
.alert("Title", isPresented: $show)
Confirmation/cancel dialog. A common UX pattern for delete confirmations.

Topic 08

UserDefaults + Codable Persistence

Keeping Data After App Restart
Swift 
// Save
func save(_ todos: [Todo]) {
    if let data = try? JSONEncoder()
        .encode(todos) {
        UserDefaults.standard
            .set(data, forKey: "todos")
    }
}

// Load
func load() -> [Todo] {
    guard let data = UserDefaults.standard
        .data(forKey: "todos"),
        let todos = try? JSONDecoder()
            .decode([Todo].self, from: data)
    else { return [] }
    return todos
}
Think About It
  • Why can't you store [Todo] directly in UserDefaults?
  • What conversions do JSONEncoder and JSONDecoder perform respectively?
  • When should the save happen to ensure data persists after an app restart?
JSONEncoder / JSONDecoder
Convert Codable types to Data and store in UserDefaults. Reverse the process to load.
.onAppear / .onChange
Load when the View appears, save when state changes — the auto-save pattern.
Empty State
Show a placeholder view when items.isEmpty. Use ContentUnavailableView (iOS 17+).

2. Todo App
Swift 
struct Todo: Identifiable, Codable {
    let id: UUID
    var title: String
    var isCompleted: Bool

    init(title: String) {
        self.id = UUID()
        self.title = title
        self.isCompleted = false
    }
}

struct TodoListView: View {
    @State private var todos: [Todo] = []
    @State private var newTitle = ""

    var body: some View {
        NavigationStack {
            List {
                ForEach($todos) { $todo in
                    HStack {
                        Image(systemName: todo.isCompleted
                            ? "checkmark.circle.fill"
                            : "circle")
                            .foregroundStyle(
                                todo.isCompleted
                                    ? .blue : .gray)
                            .onTapGesture {
                                todo.isCompleted.toggle()
                                save()
                            }
                        Text(todo.title)
                            .strikethrough(
                                todo.isCompleted)
                    }
                }
                .onDelete { offsets in
                    todos.remove(atOffsets: offsets)
                    save()
                }
            }
            .navigationTitle("Todo")
            .toolbar {
                ToolbarItem(placement: .bottomBar) {
                    HStack {
                        TextField("New todo",
                            text: $newTitle)
                        Button("Add") {
                            guard !newTitle.isEmpty
                                else { return }
                            todos.append(
                                Todo(title: newTitle))
                            newTitle = ""
                            save()
                        }
                    }
                }
            }
            .onAppear { load() }
        }
    }

    func save() {
        if let data = try? JSONEncoder()
            .encode(todos) {
            UserDefaults.standard
                .set(data, forKey: "todos")
        }
    }

    func load() {
        guard let data = UserDefaults.standard
            .data(forKey: "todos"),
            let saved = try? JSONDecoder()
                .decode([Todo].self, from: data)
        else { return }
        todos = saved
    }
}
3. Settings Screen
Swift 
struct SettingsView: View {
    @State private var isDarkMode = false
    @State private var fontSize: Double = 16
    @State private var language = "Korean"
    let languages = ["Korean", "English", "Japanese"]

    var body: some View {
        NavigationStack {
            Form {
                Section("Display") {
                    Toggle("Dark Mode",
                        isOn: $isDarkMode)
                    VStack(alignment: .leading) {
                        Text("Font Size: \(Int(fontSize))")
                        Slider(value: $fontSize,
                            in: 12...28, step: 1)
                    }
                }
                Section("Language") {
                    Picker("Select Language",
                        selection: $language) {
                        ForEach(languages, id: \.self) {
                            Text($0)
                        }
                    }
                }
                Section {
                    PreviewCard(
                        isDarkMode: $isDarkMode,
                        fontSize: $fontSize)
                }
            }
            .navigationTitle("Settings")
        }
    }
}

struct PreviewCard: View {
    @Binding var isDarkMode: Bool
    @Binding var fontSize: Double

    var body: some View {
        Text("Preview Text")
            .font(.system(size: fontSize))
            .padding()
            .frame(maxWidth: .infinity)
            .background(isDarkMode ? .black : .white)
            .foregroundStyle(isDarkMode ? .white : .black)
            .cornerRadius(12)
    }
}
