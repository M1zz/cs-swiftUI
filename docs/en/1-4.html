Topic 01

let vs var · Type Inference

Constants, variables, and when to specify types
Swift 
let name = "DevRi"           // String inferred
let age: Int = 10             // Explicit type
var score = 0                 // Mutable

score += 10
// name = "otherName"         // ❌ Compile error!

print("Name: \(name)")
print("Score: \(score)")
Think About It
  • What error do you get when you try to change a variable declared with let?
  • How does Swift infer the type as Int in let x = 42?
  • When exactly do you need to specify types explicitly?
let — Constant (immutable)
Cannot be changed once assigned. The compiler prevents unnecessary mutations. Default to let and switch to var when needed
Type Inference vs Explicit
let x = 42 (Int inferred) vs let x: Int = 42. Explicit needed when the type is ambiguous or you want a different type
Topic 02

Optional Deep Dive

nil · if let · guard let · ?? · !
Swift 
var nickname: String? = nil

// if let unwrapping
if let name = nickname {
    print("Nickname: \(name)")
} else {
    print("No nickname")
}

// ?? nil coalescing
let display = nickname ?? "Anonymous"
print("Display: \(display)")

// Optional chaining
let count = nickname?.count
print("Length: \(count)")
Think About It
  • Are String and String? completely different types? What is their relationship?
  • How do if let and guard let differ in the scope where the unwrapped value is available?
  • What is the exact condition under which ! force unwrapping causes a crash?
  • In user?.profile?.name, what is the result if any value in the chain is nil?
What is Optional
String? = "may or may not have a value". Optional is an enum: case some(Wrapped), case none
if let name = optionalName
Executes the block if a value exists. Otherwise else. The unwrapped value is only available inside the block
guard let name = optionalName else { return }
Early return if nil. The unwrapped value remains available in subsequent code. Reduces nesting
?. Optional Chaining
user?.profile?.name — if any part is nil, the whole expression is nil. Safe without crashing
?? Nil Coalescing · ! Force Unwrapping
name ?? "Anonymous" — default value when nil. ! — crashes if nil. Only use ! when you are 100% certain it is not nil
Topic 03

Array · Dictionary · Set

When to use which
Swift 
// Array
var fruits = ["Apple", "Banana", "Grape"]
fruits.append("Strawberry")
print(fruits[0])
print(fruits.count)

// Dictionary
let scores: [String: Int] = [
    "Math": 90,
    "English": 85,
]
print(scores["Math"] ?? 0)

// Set
let tags: Set = ["Swift", "iOS", "Swift"]
print(tags.count)  // Duplicates removed
Think About It
  • What error occurs when you access array[5] but the array only has 3 elements?
  • When you access a Dictionary with a non-existent key, does it crash or return nil?
  • Why is Set's contains() faster than Array's contains()?
Array — Ordered collection
Access by index. Check bounds with indices.contains(i) before accessing. Use array.indices for safe iteration
Dictionary — Key-value pairs
dict["key"] always returns Optional. The key might not exist. Use dict["key", default: 0] for a default value
Set — Collection without duplicates
Unordered. contains() is O(1). Much faster than Array for membership checks. Requires Hashable
Topic 04

Higher-Order Functions — map · filter · forEach

Declarative instead of loops
Swift 
let numbers = [1, 2, 3, 4, 5]

// map — Transform
let doubled = numbers.map { $0 * 2 }
print(doubled)

// filter — Filter
let evens = numbers.filter { $0 % 2 == 0 }
print(evens)

// compactMap — Remove nils
let strings = ["1", "a", "3"]
let nums = strings.compactMap { Int($0) }
print(nums)

// reduce — Accumulate
let sum = numbers.reduce(0, +)
print(sum)
Think About It
  • What is the difference between map and forEach? Both iterate over elements, so why distinguish them?
  • How does compactMap differ from map? When should you use it?
  • In reduce(0, +), what are 0 and + respectively?
map — Transform
names.map { $0.uppercased() } — transforms each element and returns a new array
filter — Filter
items.filter { $0.isCompleted } — returns a new array with only matching elements
compactMap — Remove Optionals
map + nil removal. strings.compactMap { Int($0) } — filters out nil results from failed conversions
reduce — Accumulate
items.reduce(0) { $0 + $1.price } — calculates a cumulative value
Topic 05

Closure Syntax Step-by-Step Shortening

Full form → Trailing → Shorthand arguments
Swift 
let names = ["Charlie", "Alice", "Bob"]

// 1. Full form
let a = names.sorted(by: {
    (a: String, b: String) -> Bool in
    return a < b
})

// 2. Type inference
let b = names.sorted(by: { a, b in a < b })

// 3. Shorthand arguments
let c = names.sorted(by: { $0 < $1 })

// 4. Trailing closure
let d = names.sorted { $0 < $1 }

print(d)
Think About It
  • What is the shortest form of { (a: String, b: String) -> Bool in return a < b }?
  • What do $0 and $1 mean? If there's also a $2, how many parameters are there?
  • What exactly is a trailing closure? Why does the Button { } syntax work?
Full syntax
{ (name: String) -> String in return name.uppercased() }
Shortened with type inference
{ name in name.uppercased() } — type and return omitted
Shorthand arguments $0 · $1
{ $0.uppercased() } — parameter names omitted too. Numbered in order: $0, $1, ...
Trailing closure
When the last parameter is a closure, it goes outside the parentheses. Button { } / .onAppear { } use this form
Topic 06

[weak self] Capture List

When it's needed and when it's not
Swift 
class DataLoader {
    var data: String?

    func load() {
        // ✅ [weak self] needed
        URLSession.shared.dataTask(
            with: url
        ) { [weak self] data, _, _ in
            guard let self else { return }
            self.data = String(
                data: data!, encoding: .utf8
            )
        }.resume()
    }
}

// ❌ Not needed in SwiftUI Views
struct MyView: View {
    var body: some View {
        Button("Tap") {
            // struct, so no retain cycle
            print("no [weak self] needed")
        }
    }
}
Think About It
  • What is a retain cycle and why does it cause memory leaks?
  • When you add [weak self], what state can self be in?
  • Why is [weak self] unnecessary in SwiftUI Views?
Closures capture values
When a closure references self, it creates a strong capture. If self also references the closure, a retain cycle occurs
When [weak self] is needed
In escaping closures (async callbacks, timers, etc.) that reference self. [weak self] guard let self else { return }
Not needed in SwiftUI
SwiftUI Views are structs — not classes, so no retain cycles. Also unnecessary in map { }, filter { }, etc.

1. Optional Practice Playground
Swift 
var name: String? = "Kim"
var age: Int? = nil
var tags: [String]? = ["Swift", "iOS"]

// if let
if let name {
    print("Name: \(name)")       // Name: Kim
}

// guard let
func printAge() {
    guard let age else {
        print("No age info")     // Printed
        return
    }
    print("Age: \(age)")
}
printAge()

// ?? nil coalescing
let displayAge = age ?? 0
print("Age: \(displayAge)")     // Age: 0

// Optional chaining
let count = tags?.count
print("Tag count: \(count)")   // Optional(2)

let first = tags?.first?.uppercased()
print("First tag: \(first)")   // Optional("SWIFT")
2. Process Data with Higher-Order Functions
Swift 
struct Student {
    let name: String
    let score: Int
}

let students = [
    Student(name: "Alice", score: 92),
    Student(name: "Bob", score: 75),
    Student(name: "Charlie", score: 88),
    Student(name: "Diana", score: 64),
    Student(name: "Eve", score: 95),
]

// map — Extract names only
let names = students.map { $0.name }
print(names)
// ["Alice", "Bob", "Charlie", "Diana", "Eve"]

// filter — Score 80 or above
let topStudents = students.filter { $0.score >= 80 }
print(topStudents.map { $0.name })
// ["Alice", "Charlie", "Eve"]

// reduce — Average score
let total = students.reduce(0) { $0 + $1.score }
let average = Double(total) / Double(students.count)
print("Average: \(average)")  // Average: 82.8
3. Closure Step-by-Step Shortening
Swift 
let numbers = [5, 2, 8, 1, 9, 3]

// === sorted ===
// 1) Full form
numbers.sorted(by: { (a: Int, b: Int) -> Bool in
    return a < b
})
// 2) Type inference
numbers.sorted(by: { a, b in a < b })
// 3) Shorthand arguments
numbers.sorted(by: { $0 < $1 })
// 4) Trailing closure
numbers.sorted { $0 < $1 }

// === filter ===
// 1) Full form
numbers.filter({ (n: Int) -> Bool in
    return n > 3
})
// 2) Type inference
numbers.filter({ n in n > 3 })
// 3) Shorthand arguments
numbers.filter({ $0 > 3 })
// 4) Trailing closure
numbers.filter { $0 > 3 }

// === map ===
// 1) Full form
numbers.map({ (n: Int) -> String in
    return "\(n) pts"
})
// 2) Type inference
numbers.map({ n in "\(n) pts" })
// 3) Shorthand arguments
numbers.map({ "\($0) pts" })
// 4) Trailing closure
numbers.map { "\($0) pts" }
