Step 00
시작 전에 —
무엇을 배우는가
커리큘럼 어디와 연결되는지 먼저 파악하고 시작한다.
@Observable
iOS 17+의 새 관찰 매크로. class에 붙이면 프로퍼티가 바뀔 때 구독 중인 뷰가 자동으로 다시 그려진다.
.environment()
상위 뷰에서 객체를 주입하면 하위 어느 뷰에서든 @Environment로 꺼내 쓸 수 있다. 전역 상태 공유에 적합.
reduce
배열을 단일 값으로 합산. 장바구니 총 금액 계산에 딱 맞는 고차 함수.
.searchable
NavigationStack 안에서 한 줄로 검색 바를 추가. 바인딩된 텍스트로 목록을 실시간 필터링한다.
선수 조건
Stage 1-2 (@State, Button, List), Stage 1-3 (NavigationStack, Sheet, @Environment) 을 먼저 완료해야 합니다.
완성 앱 미리보기

실제 쇼핑 앱처럼 동작하는 두 화면을 만듭니다:

  • 상품 목록: 카테고리 필터 칩 + 검색 바 + 수량 조절 Stepper
  • 장바구니: 담긴 상품 목록 + 스와이프 삭제 + 총 금액 표시
  • 상품을 담으면 장바구니 탭 배지 숫자가 즉시 업데이트
  • 결제 전 confirmationDialog로 재확인
공부하는 방법
완성 코드를 먼저 실행해서 동작을 확인한 뒤, 빈 프로젝트에서 이 가이드를 따라 직접 처음부터 만들어보세요. 막히면 완성 코드를 참고하되, 복붙은 피하고 직접 타이핑하는 것이 훨씬 효과적입니다.
Step 01
모델 설계
Product, CartItem struct와 CartStore @Observable class를 정의한다.
데이터 모델 구조
①
Product — 판매 상품
ID, 이름, 가격, 카테고리, 이미지 이름. 변하지 않는 정보이므로 struct + let으로 정의.
②
CartItem — 장바구니 항목
Product를 담고 수량(quantity)을 추가. subtotal 계산 프로퍼티 포함.
③
CartStore — 상태 저장소
items 배열을 소유하고 담기/수량변경/삭제 메서드를 제공하는 @Observable class.
Product & CartItem
struct Product: Identifiable {
    let id = UUID()
    let name: String
    let price: Int
    let category: String
    let imageName: String
}

struct CartItem: Identifiable {
    let id = UUID()
    let product: Product
    var quantity: Int

    var subtotal: Int { product.price * quantity }
}
CartStore @Observable class
@Observable
class CartStore {
    var items: [CartItem] = []

    var totalPrice: Int {
        items.reduce(0) { $0 + $1.subtotal }
    }

    func add(_ product: Product) {
        if let i = items.firstIndex(where: { $0.product.id == product.id }) {
            items[i].quantity += 1
        } else {
            items.append(CartItem(product: product, quantity: 1))
        }
    }

    func delete(at offsets: IndexSet) {
        items.remove(atOffsets: offsets)
    }
}
struct vs class 선택 기준
Product와 CartItem은 값 타입(struct)으로 충분합니다. 하지만 CartStore는 여러 뷰에서 "같은 인스턴스"를 공유해야 하므로 참조 타입(class) + @Observable 조합을 사용합니다.
Step 02
앱 진입점과 환경 주입
.environment(store)로 CartStore를 전체 뷰 트리에 공유한다.
왜 .environment를 쓰는가?

상품 목록 화면과 장바구니 화면은 화면 계층상 형제 관계입니다. @State로 상위 뷰에 두고 @Binding으로 내려보내도 되지만, 화면이 깊어질수록 바인딩 체인이 길어집니다. .environment를 쓰면 중간 뷰를 거치지 않고 어느 깊이에서든 꺼내 쓸 수 있습니다.

CartApp — 진입점
@main
struct CartApp: App {
    // 앱 생명주기 동안 단 하나의 인스턴스 유지
    @State private var store = CartStore()

    var body: some Scene {
        WindowGroup {
            ContentView()
                .environment(store)   // 전체 뷰 트리에 주입
        }
    }
}
하위 뷰에서 꺼내 쓰기
struct ProductListView: View {
    // @Observable 객체는 타입으로 꺼낸다 (key path 불필요)
    @Environment(CartStore.self) private var store

    var body: some View {
        Button("담기") {
            store.add(product)
        }
    }
}

struct CartView: View {
    @Environment(CartStore.self) private var store

    var body: some View {
        Text("총 \(store.totalPrice)원")
    }
}
@Environment vs @EnvironmentObject
iOS 17 이전에는 ObservableObject + @EnvironmentObject를 사용했습니다. iOS 17+에서는 @Observable + @Environment(Type.self) 조합이 더 간결하고 성능도 좋습니다. 이 프로젝트는 iOS 17+ 타겟을 기준으로 합니다.

Step 03
상품 목록 화면
List, .searchable, 카테고리 필터 칩, QuantityStepper를 구현한다.
.searchable 검색 바

NavigationStack 안에서 .searchable modifier 한 줄로 검색 바가 생깁니다:

struct ProductListView: View {
    @Environment(CartStore.self) private var store
    @State private var searchText = ""
    @State private var selectedCategory: String? = nil

    private var filtered: [Product] {
        SampleData.products
            .filter { selectedCategory == nil || $0.category == selectedCategory }
            .filter { searchText.isEmpty || $0.name.contains(searchText) }
    }

    var body: some View {
        NavigationStack {
            VStack(spacing: 0) {
                categoryFilterChips
                List(filtered) { product in
                    ProductRow(product: product)
                }
            }
            .navigationTitle("상품 목록")
            .searchable(text: $searchText, prompt: "상품 검색")
        }
    }
}
카테고리 필터 칩
private var categoryFilterChips: some View {
    ScrollView(.horizontal, showsIndicators: false) {
        HStack(spacing: 8) {
            ForEach(categories, id: \.self) { cat in
                Button(cat) {
                    selectedCategory = selectedCategory == cat ? nil : cat
                }
                .padding(.horizontal, 14)
                .padding(.vertical, 6)
                .background(selectedCategory == cat ? Color.orange : Color(.systemGray6))
                .foregroundStyle(selectedCategory == cat ? .white : .primary)
                .clipShape(Capsule())
                .font(.footnote.weight(.medium))
            }
        }
        .padding(.horizontal)
        .padding(.vertical, 8)
    }
}
QuantityStepper — 수량 조절
struct QuantityStepper: View {
    let product: Product
    @Environment(CartStore.self) private var store

    private var quantity: Int {
        store.items.first(where: { $0.product.id == product.id })?.quantity ?? 0
    }

    var body: some View {
        HStack(spacing: 12) {
            Button { store.decrease(product) } label: {
                Image(systemName: "minus.circle.fill")
            }
            Text("\(quantity)").frame(minWidth: 20)
            Button { store.add(product) } label: {
                Image(systemName: "plus.circle.fill")
            }
        }
        .foregroundStyle(.orange)
    }
}
Step 04
장바구니 화면
.onDelete, reduce로 총 금액을 계산하고 confirmationDialog로 결제를 확인한다.
CartView 전체 구조
struct CartView: View {
    @Environment(CartStore.self) private var store
    @State private var showCheckout = false

    var body: some View {
        NavigationStack {
            List {
                ForEach(store.items) { item in
                    CartRow(item: item)
                }
                .onDelete { store.delete(at: $0) }
            }
            .navigationTitle("장바구니")
            .safeAreaInset(edge: .bottom) {
                checkoutBar
            }
        }
    }
}
총 금액 — reduce
// CartStore 안에서 계산 프로퍼티로 정의
var totalPrice: Int {
    items.reduce(0) { $0 + $1.subtotal }
}

// 뷰에서 사용
Text("합계 \(store.totalPrice)원")
    .font(.headline)
.onDelete — 스와이프 삭제
ForEach(store.items) { item in
    CartRow(item: item)
}
.onDelete { offsets in
    store.delete(at: offsets)
}
결제 확인 — confirmationDialog
Button("결제하기") {
    showCheckout = true
}
.confirmationDialog(
    "총 \(store.totalPrice)원을 결제하시겠습니까?",
    isPresented: $showCheckout,
    titleVisibility: .visible
) {
    Button("결제", role: .destructive) {
        store.checkout()
    }
    Button("취소", role: .cancel) {}
}
safeAreaInset(edge: .bottom)
결제 바를 List 위에 고정하면서 List 스크롤이 결제 바에 가려지지 않도록 safeAreaInset을 사용합니다. padding을 직접 계산하지 않아도 됩니다.
