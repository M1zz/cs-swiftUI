Step 00
SwiftData vs UserDefaults
언제 어떤 저장 방식을 쓰는지 판단 기준을 이해한다.
UserDefaults
설정값, 소량 데이터. 수동 Codable 직렬화 필요. 관계(Relationship) 표현 불가.
SwiftData
수백~수만 건 데이터, 관계형 데이터. @Model 선언만으로 자동 저장. @Query로 자동 조회.
@Model
class에 붙이면 SwiftData가 자동으로 데이터베이스 스키마를 생성. Codable 불필요.
@Query
뷰에서 데이터를 자동으로 가져오고, 변경 시 자동으로 리렌더링. sort/filter 지원.
가계부에 SwiftData가 적합한 이유
거래 데이터는 수백 건 이상이 될 수 있고, 날짜별/카테고리별 필터가 필요합니다. UserDefaults는 전체를 한 번에 로드/저장하므로 데이터가 많아지면 성능이 저하됩니다. SwiftData는 필요한 데이터만 선택적으로 쿼리할 수 있습니다.
Step 01
Transaction 모델
@Model 클래스 하나로 데이터베이스 스키마가 자동 생성된다.
@Model 선언
import SwiftData

@Model
class Transaction {
    var id: UUID
    var title: String
    var amount: Double    // 양수=수입, 음수=지출
    var category: String
    var date: Date

    init(title: String, amount: Double, category: String, date: Date) {
        self.id = UUID()
        self.title = title
        self.amount = amount
        self.category = category
        self.date = date
    }

    var isIncome: Bool { amount > 0 }
    var absAmount: Double { abs(amount) }
}
앱 진입점 — ModelContainer 설정
@main
struct BudgetApp: App {
    var body: some Scene {

        // 이 한 줄이 데이터베이스 파일을 생성하고

    }
}

저장 — 수입은 양수, 지출은 음수
private func save() {
    let finalAmount = isIncome ? amount : -amount
    let t = Transaction(
        title: title,
        amount: finalAmount,    // 부호로 수입/지출 구분
        category: selectedCategory,
        date: date
    )
    context.insert(t)
    dismiss()
}
금액 입력 처리

Double(amountText.replacingOccurrences(of: ",", with: "")) ?? 0
Step 03
거래 목록
@Query로 데이터를 자동으로 가져오고 날짜별로 그룹핑한다.
@Query — 자동 조회 + 정렬
// 날짜 역순으로 모든 거래 자동 조회
@Query(sort: \Transaction.date, order: .reverse)
private var transactions: [Transaction]
// 데이터가 추가/삭제되면 자동으로 뷰가 갱신됨
날짜별 그룹핑
private var grouped: [(String, [Transaction])] {
    let dict = Dictionary(grouping: filtered) { dateKey($0.date) }
    return dict.sorted { $0.key > $1.key }  // 최신 날짜 먼저
}

private func dateKey(_ date: Date) -> String {
    let f = DateFormatter()
    f.dateFormat = "yyyy년 M월 d일 (E)"
    f.locale = Locale(identifier: "ko_KR")
    return f.string(from: date)
}

Step 05 + 06
Predicate 필터 & 도전 과제
@Query에 조건을 추가하고, 예산 알림 기능을 구현한다.
@Query Predicate로 서버 측 필터
// 지출만 조회하는 @Query
@Query(filter: #Predicate { $0.amount < 0 },
       sort: \Transaction.date, order: .reverse)
private var expenses: [Transaction]

// 특정 카테고리만 조회
@Query(filter: #Predicate { $0.category == "식비" })
private var foodExpenses: [Transaction]
Predicate vs 코드 내 filter
Predicate는 데이터베이스 수준에서 필터링 → 대용량 데이터에서 성능 우수.
배열의 .filter { } 는 모든 데이터를 메모리에 로드 후 필터 → 소량 데이터에서 OK.
