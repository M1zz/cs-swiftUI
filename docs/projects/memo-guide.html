Step 00
영속성이란
무엇인가
앱을 껐다 켜도 데이터가 남는 것. 이 프로젝트의 핵심 목표.
Codable
struct를 JSON Data로 변환하거나, JSON Data를 struct로 복원한다. UserDefaults에 저장하기 위한 필수 단계.
UserDefaults
앱 설정과 소량 데이터를 키-값 쌍으로 영구 저장. JSON Data → set(data, forKey:) → data(forKey:) 패턴.
didSet 자동 저장
var memos: [Memo] = [] { didSet { save() } } — 배열이 바뀔 때마다 자동으로 UserDefaults에 저장.
@Observable
MemoStore를 @Observable로 선언하면 뷰가 store.memos 변경을 자동으로 감지해 리렌더링.
데이터 흐름
뷰 → store.add() → memos 배열 변경 → didSet → save() → UserDefaults
앱 시작 → MemoStore.init() → load() → UserDefaults → memos 복원

이 프로젝트를 완성하면 앱을 껐다 켜도 데이터가 유지되는 경험을 처음으로 하게 됩니다. Stage 2-1에서 SwiftData를 배우기 전, UserDefaults 수동 저장으로 영속성의 원리를 이해하는 데 목적이 있습니다.

완성 앱 기능
메모 추가 / 편집 / 삭제 (CRUD) · 날짜순/이름순 정렬 · 제목+본문 검색 (.searchable) · 앱 재시작 후 데이터 유지
Step 01
Memo 모델 설계
Codable + Identifiable을 채택한 struct를 만든다.
왜 struct인가?

메모 데이터는 뷰끼리 공유할 필요가 없는 값 타입입니다. struct를 사용하면 배열에서 수정할 때 새 복사본이 만들어지므로, SwiftUI의 상태 관리와 자연스럽게 맞습니다.

struct Memo: Codable, Identifiable {
    var id: UUID = UUID()    // List/ForEach 식별자
    var title: String
    var body: String
    var updatedAt: Date = Date()

    // 목록에 보여줄 첫 번째 줄 미리보기
    var preview: String {
        let firstLine = body.components(separatedBy: "\n").first ?? ""
        return firstLine.isEmpty ? "내용 없음" : firstLine
    }
}
Codable = Encodable + Decodable
Codable을 채택하면 Swift가 자동으로 프로퍼티 이름 기반의 JSON 인코딩/디코딩 코드를 생성합니다. 프로퍼티 이름이 JSON 키와 다를 때는 CodingKeys enum을 정의합니다.
Codable 동작 확인
let memo = Memo(title: "테스트", body: "내용")

// 인코딩 (Memo → Data)
let data = try? JSONEncoder().encode(memo)

// 디코딩 (Data → Memo)
if let data,
   let decoded = try? JSONDecoder().decode(Memo.self, from: data) {
    print(decoded.title)  // "테스트"
}
배열 저장

메모가 여러 개이므로 [Memo] 배열 자체를 인코딩합니다. 배열도 Codable의 요소가 Codable이면 자동으로 Codable입니다.

// 배열 인코딩
let memos: [Memo] = [...]
let data = try? JSONEncoder().encode(memos)  // OK

// 배열 디코딩
let restored = try? JSONDecoder().decode([Memo].self, from: data!)
Step 02
MemoStore
@Observable 클래스로 CRUD + UserDefaults 자동 저장을 구현한다.
didSet으로 자동 저장

memos 배열이 변경될 때마다 자동으로 저장되도록 didSet을 활용합니다.

@Observable
class MemoStore {
    private let key = "memos_v1"

    var memos: [Memo] = [] {
        didSet { save() }   // 변경 감지 → 즉시 저장
    }

    init() { load() }    // 앱 시작 시 복원
}
CRUD 메서드
// Create — 최신 순으로 맨 앞에 추가
func add(title: String = "새 메모") {
    memos.insert(Memo(title: title, body: ""), at: 0)
}

// Update
func update(_ memo: Memo) {
    guard let i = memos.firstIndex(where: { $0.id == memo.id }) else { return }
    var updated = memo
    updated.updatedAt = Date()
    memos[i] = updated    // struct 교체 → didSet 발동 → 저장
}

// Delete
func delete(at offsets: IndexSet) {
    memos.remove(atOffsets: offsets)
}
저장 / 불러오기
private func save() {
    guard let data = try? JSONEncoder().encode(memos) else { return }
    UserDefaults.standard.set(data, forKey: key)
}

private func load() {
    guard
        let data = UserDefaults.standard.data(forKey: key),
        let saved = try? JSONDecoder().decode([Memo].self, from: data)
    else { return }
    memos = saved
}
key 버전 관리 — "memos_v1"
나중에 Memo struct에 새 프로퍼티를 추가하면 기존 저장 데이터와 호환이 안 될 수 있습니다. key를 "memos_v2"로 바꾸면 깔끔하게 초기화됩니다.

Step 04
메모 편집 화면
로컬 복사본(@State)으로 편집하고 저장 버튼에서 store.update()를 호출한다.
편집의 핵심 — 로컬 복사본

편집 화면에서 바로 store.memos를 수정하면, 사용자가 "취소"를 눌러도 이미 변경이 반영됩니다. @State 로컬 복사본으로 편집하고, 저장 버튼에서만 store에 반영합니다.

struct MemoEditView: View {
    @State private var title: String
    @State private var body: String
    private let originalMemo: Memo

    // init에서 @State 초기값 설정
    init(memo: Memo) {
        self.originalMemo = memo
        _title = State(initialValue: memo.title)
        _body  = State(initialValue: memo.body)
    }
}
_title = State(initialValue:) 패턴
init에서 @State를 초기화할 때는 언더스코어(_title)로 접근해서 State 래퍼 자체를 초기화해야 합니다. 일반 할당(self.title = ...)은 작동하지 않습니다.
저장 버튼
Button("저장") {
    var updated = originalMemo
    updated.title = title.isEmpty ? "새 메모" : title
    updated.body  = body
    store.update(updated)   // store → didSet → save()
    dismiss()
}
TextEditor vs TextField
  • TextField: 한 줄 입력. 제목에 사용.
  • TextEditor: 여러 줄 입력. 본문에 사용. 자체 배경색이 있어 .scrollContentBackground(.hidden)으로 제거 가능.
TextEditor(text: $body)
    .scrollContentBackground(.hidden) // 기본 흰 배경 제거
    .background(.clear)
