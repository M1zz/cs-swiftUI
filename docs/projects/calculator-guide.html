완성 앱 미리보기

Apple 기본 계산기와 동일한 UI를 구현합니다:

  • 숫자 버튼 (0~9), 연산자 버튼 (+, −, ×, ÷, =)
  • 기능 버튼 (AC, +/−, %)
  • 0 버튼은 두 칸 너비 (isWide)
  • 숫자가 길어지면 폰트 크기 자동 축소
  • 연속 연산 지원 (3 + 5 × 2 등)
공부하는 방법
완성 코드를 먼저 실행해서 동작을 확인한 뒤, 빈 프로젝트에서 이 가이드를 따라 직접 처음부터 만들어보세요. 막히면 완성 코드를 참고하되, 복붙은 피하고 직접 타이핑하는 것이 훨씬 효과적입니다.
Step 01
앱 구조 설계
화면을 분해하고 데이터 모델을 먼저 정의한다.
화면 분해

Apple 계산기를 보면 화면은 크게 두 부분으로 나뉩니다:

①
디스플레이 영역
숫자가 표시되는 오른쪽 정렬 텍스트. 숫자가 길면 글자가 작아진다.
②
버튼 그리드 영역
4열 × 5행. 단, 마지막 행의 0 버튼은 두 칸 너비.
파일 구조
// Xcode 프로젝트 파일 구조
Calculator/
├── CalculatorApp.swift    // 앱 진입점 (@main)
├── ContentView.swift      // 메인 화면 (UI 담당)
├── CalcButton.swift       // 버튼 모델 (enum)
└── CalculatorLogic.swift  // 계산 로직 (@Observable)
CalcButton enum 설계

버튼을 문자열로 관리하면 실수가 생깁니다. enum으로 타입 안전하게 표현합니다:

enum CalcButton: String {
    // rawValue = 버튼에 표시할 텍스트
    case zero = "0", one = "1" /* ... */
    case plus = "+", minus = "−"
    case ac = "AC", equals = "="

    // 버튼 색상도 enum에서 직접 정의
    var backgroundColor: Color {
        switch self {
        case .ac, .plusMinus, .percent: return .gray
        case .plus, .minus, .multiply, .divide, .equals: return .orange
        default: return Color(.darkGray)
        }
    }
}
왜 enum인가?
String으로 버튼을 식별하면 오타가 나도 컴파일러가 잡을 수 없습니다. enum을 쓰면 존재하지 않는 케이스를 쓸 때 컴파일 에러가 납니다. switch의 exhaustive check도 큰 장점입니다.
Step 02
버튼 UI 구성
LazyVGrid로 4열 버튼 그리드를 만들고 스타일을 입힌다.
버튼 레이아웃 데이터

버튼 배치를 2D 배열로 정의합니다. 뷰 코드에 직접 하드코딩하지 않는 게 핵심입니다:

let buttonLayout: [[CalcButton]] = [
    [.ac,    .plusMinus, .percent, .divide  ],
    [.seven, .eight,    .nine,   .multiply],
    [.four,  .five,     .six,    .minus   ],
    [.one,   .two,      .three,  .plus    ],
    [.zero,             .dot,    .equals  ],  // zero는 두 칸
]
ContentView 구조
struct ContentView: View {
    @State private var logic = CalculatorLogic()

    var body: some View {
        ZStack {
            Color.black.ignoresSafeArea()
            VStack(spacing: 12) {
                Spacer()
                displayView     // 숫자 표시
                buttonGrid      // 버튼 그리드
            }
            .padding(12)
        }
    }
}
버튼 그리드 (행 단위 HStack)

LazyVGrid 대신 행 단위 HStack을 중첩하는 방식을 씁니다. 0 버튼의 두 칸 처리가 더 직관적이기 때문입니다:

private var buttonGrid: some View {
    VStack(spacing: 12) {
        ForEach(buttonLayout, id: \.self) { row in
            HStack(spacing: 12) {
                ForEach(row, id: \.self) { button in
                    CalcButtonView(button: button, size: buttonSize) {
                        logic.tap(button)
                    }
                }
            }
        }
    }
}
개별 버튼 뷰
struct CalcButtonView: View {
    let button: CalcButton
    let size: CGFloat
    let action: () -> Void

    // 0 버튼이면 두 칸 너비
    private var width: CGFloat {
        button.isWide ? size * 2 + 12 : size
    }

    var body: some View {
        Button(action: action) {
            Text(button.rawValue)
                .font(.system(size: 32, weight: .medium))
                .foregroundStyle(button.foregroundColor)
                .frame(width: width, height: size)
                .background(button.backgroundColor)
                .clipShape(Capsule())
        }
    }
}
buttonSize 계산
버튼 크기는 화면 너비에서 동적으로 계산합니다:
buttonSize = (UIScreen.main.bounds.width - 5 × gap) / 4
기기 크기에 상관없이 항상 4등분이 됩니다.
Step 03
연산 로직
@Observable 클래스로 로직과 뷰를 분리한다.
왜 로직을 뷰에서 분리하는가?

ContentView에 모든 계산 로직을 넣으면 뷰 코드가 200줄 이상이 됩니다. 로직을 별도 클래스로 분리하면:

  • 뷰는 "어떻게 보여줄지"만 담당
  • 로직 클래스는 "어떻게 계산할지"만 담당
  • 테스트와 유지보수가 쉬워짐
CalculatorLogic 핵심 상태
@Observable
class CalculatorLogic {
    // 뷰가 관찰하는 단 하나의 프로퍼티
    var displayText: String = "0"

    // 내부 상태 (뷰에서 볼 필요 없음 → private)
    private var currentValue: Double = 0
    private var previousValue: Double = 0
    private var currentOperator: Operator?
    private var shouldResetDisplay = false
}
버튼 탭 → 분기
func tap(_ button: CalcButton) {
    switch button {
    case .ac:                  reset()
    case .plusMinus:          toggleSign()
    case .percent:             applyPercent()
    case .plus:               setOperator(.add)
    case .minus:              setOperator(.subtract)
    case .multiply:           setOperator(.multiply)
    case .divide:             setOperator(.divide)
    case .equals:             calculate()
    case .dot:                appendDot()
    default:                  appendDigit(button.rawValue)
    }
}
연산자 처리 — shouldResetDisplay 플래그

연산자를 누른 뒤 다음 숫자를 입력할 때 화면을 초기화해야 합니다. 플래그 하나로 제어합니다:

private func appendDigit(_ digit: String) {
    if shouldResetDisplay {
        displayText = digit       // 화면 초기화
        shouldResetDisplay = false
    } else {
        // "0" 상태면 그냥 교체, 아니면 뒤에 붙임
        displayText = displayText == "0" ? digit : displayText + digit
    }
    currentValue = Double(displayText) ?? 0
}
계산 실행
private func calculate() {
    guard let op = currentOperator else { return }
    let result: Double
    switch op {
    case .add:      result = previousValue + currentValue
    case .subtract: result = previousValue - currentValue
    case .multiply: result = previousValue * currentValue
    case .divide:   result = previousValue / currentValue
    }
    displayText = formatted(result)
    shouldResetDisplay = true
}
Step 04
엣지 케이스 처리 & 완성
실제 계산기처럼 동작하도록 예외 상황을 처리한다.
처리해야 할 엣지 케이스
①
0으로 나누기
currentValue == 0일 때 나누기를 누르면 앱이 크래시합니다. guard 또는 삼항 연산자로 방어합니다.
case .divide:
    result = currentValue == 0 ? 0 : previousValue / currentValue
②
소수점 중복 입력
"1.2." 처럼 소수점이 두 번 찍히지 않도록 방어합니다.
private func appendDot() {
    if !displayText.contains(".") {
        displayText += "."
    }
}
③
결과 포맷 (1.0 → 1)
3 + 5 = 8.0 이 아니라 8로 표시되어야 합니다. 정수면 소수점을 생략합니다.
private func formatted(_ value: Double) -> String {
    if value.truncatingRemainder(dividingBy: 1) == 0 {
        return String(Int(value))   // 정수면 Int로 변환
    }
    return String(value)          // 소수면 그대로
}
④
디스플레이 폰트 자동 축소
숫자가 10자리 이상이면 글자를 작게 만들어 화면을 넘치지 않게 합니다.
private var displayFontSize: CGFloat {
    logic.displayText.count > 9 ? 56 : 88
}

Text(logic.displayText)
    .font(.system(size: displayFontSize, weight: .thin))
    .minimumScaleFactor(0.4)  // 더 줄어들 여지
