Top-Down
먼저 만들고, 문제를 만나고, 그때 배운다. 기초를 다 배우고 시작하는 방식은 동기를 죽인다.
💢
Problem-Driven
각 챕터는 실제 개발 중 마주치는 문제에서 시작한다. 문제가 없으면 개념도 없다.
📱
App-First
모든 예시는 iOS 앱 개발 맥락 안에 있다. 추상적인 예제가 아닌 실제 코드로 배운다.

🛠
일단 만든다
아무것도 없다 → 화면에 뭔가를 띄우고 싶다
SwiftUI @State @Binding NavigationStack Optional Closure
→
STAGE 2 준비 중
🐢
"왜 이렇게 느려?"
목록에 데이터가 많아지니 스크롤이 버벅인다
Big-O Array vs Set Heap Trie 이진 탐색
STAGE 3 준비 중
🍝
코드가 스파게티가 됐다
기능이 늘자 파일이 1000줄 / 수정하면 다른 곳이 깨진다
OOP SOLID Protocol MVVM Clean Arch
STAGE 4 준비 중
💀
앱이 죽는다
EXC_BAD_ACCESS / 메모리 경고 / deinit이 불리지 않는다
ARC weak/unowned Instruments 에러 처리
STAGE 5 준비 중
🌐
서버 데이터가 필요하다
API를 호출해서 JSON을 파싱하고 싶다
HTTP URLSession Codable CoreData CloudKit
STAGE 6 준비 중
🧊
UI가 버벅이고 앱이 멈춘다
네트워크 요청 중 화면이 얼어붙는다
GCD async/await Actor Combine
STAGE 7 준비 중
🧪
내 코드가 맞는지 모르겠다
배포 후 버그 발견 / 수정하면 다른 곳이 깨질까 두렵다
XCTest TDD CI/CD Fastlane StoreKit 2
STAGE 8 준비 중
🔭
더 깊이 알고 싶다
왜 이렇게 동작하는지 근본 원리가 궁금하다
CPU 구조 OS 원리 컴파일러 Swift Macros 선형대수
